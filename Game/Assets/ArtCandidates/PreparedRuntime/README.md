# Prepared Runtime Candidates

These static sprites are loaded automatically by `GearScavengerGame.cs` while
running from the Unity editor:

- `player.png`: Tyst ranged robot, recolored with cyan energy accents.
- `enemy_chaser.png`: Robot Warfare Spider, red role accent.
- `enemy_drone.png`: Robot Warfare Hornet, amber role accent.
- `enemy_support.png`: Robot Warfare Scarab, green role accent.
- `enemy_bulwark.png`: Robot Warfare Centipede, blue armored role accent.
- `enemy_artillery.png`: Robot Warfare Wasp, purple artillery role accent.
- `enemy_boss.png`: Mechs Assets grey mech.
- `floor_a.png` / `floor_b.png`: rusted steel and grated mechanical floor.
- `wall.png`: raised industrial wall with worn hazard markings.
- `prop_a.png` / `prop_b.png`: scrap machinery.
- `crate.png`, `barrel.png`, `terminal.png`: mechanical wasteland props.
- `weapon_*.png`: a unified family of scavenged modular weapons.

Delete or rename an individual PNG to make that role fall back to the original
sprite in `Assets/Resources/GearScavenger`.

The map and weapon candidates were created specifically for Gear Scavenger to
match the selected free robot art direction.

This remains a static visual pass. The original animation frames remain in each
source package and can be integrated after the lineup is approved.
